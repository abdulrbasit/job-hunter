# Examples

Reference files showing what a fully configured Job Hunter workspace looks like.

All names, companies, metrics, and contact details are **fictional**. They exist to show the format and level of detail that makes the pipeline work well — not as templates to copy word-for-word.

## Persona

**John Doe** — Senior Backend Engineer, Hamburg.  
8 years in fintech and payments infrastructure. Targeting Staff and Senior Backend roles.  
Background in distributed systems, Go/Rust, Kafka, and high-throughput payment pipelines.

## Files

| File | What it shows |
|---|---|
| `config/job_hunter.yml` | Filled config with real titles, region, exclusions, and LLM settings |
| `profile/career_context.md` | Fully filled career context with positioning, style guidance, and evidence rules |
| `profile/resume_double_column.tex` | Filled AltaCV double-column resume |
| `profile/resume_single_column.tex` | Filled single-column resume with the same content |

## How to use

Open any file as a reference while running `/setup onboard`, `/setup context`, or `/setup resume` in your own workspace. The level of detail in the career context — especially the evidence rules section — directly improves tailoring quality.
